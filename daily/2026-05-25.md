## Notes

- PoW 和 PoS 都是共识机制，解决的是区块链如何决定新区块和交易历史的问题。
- PoW 用算力和能源成本约束作恶，典型代表是 Bitcoin。
- PoS 用质押资产和罚没机制约束作恶，当前 Ethereum 使用 PoS。
- Hardhat 和 Foundry 是智能合约开发工具链，不是链，也不是钱包。
- Hardhat 偏 JavaScript / TypeScript 生态，适合和前端 dApp、ethers.js 集成。
- Foundry 偏 Solidity / CLI 生态，适合快速编译、测试、部署和链上调试。

## Concept Map

Two separate layers:

- Consensus layer: PoW / PoS decide how blocks are proposed, validated, and finalized.
- Developer workflow layer: Hardhat / Foundry help developers compile, test, deploy, and verify smart contracts.

Minimal smart contract workflow:

1. Write Solidity contract.
2. Write tests for expected behavior and failure cases.
3. Run local tests with Hardhat or Foundry.
4. Deploy to local chain or testnet.
5. Verify contract source and inspect transactions in an explorer.
6. Record deployment address, network, transaction hash, and known risks.
