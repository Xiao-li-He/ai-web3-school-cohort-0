### Account Abstraction账户抽象

> 账户抽象的核心不是“免 gas”，而是把账户控制权从单一私钥扩展成可编程规则。

账户抽象把“账户如何验证操作、谁来付 gas、哪些权限可以自动执行”从固定的 EOA（外部拥有账户，私钥直接控制账户）模式里释放出来，让钱包更像可编程账户系统。给 Agent 一个可限制、可过期、可撤销、可审计的行动空间。

- 验证逻辑可定制：账户可以用多签、Passkey、社交恢复或模块规则验证操作。
- 支付逻辑可定制：gas 可以由用户、应用、paymaster 或其他资产承担。
- 权限可以最小化：session key 可以只允许特定合约、额度、时间和方法。

**知识点**：

- ERC-4337：以太坊生态最重要的账户抽象标准之一，用 alt mempool 和 EntryPoint 实现智能账户交易流程。
- Smart Account：由合约控制的账户，可以把权限、恢复、批量执行和策略写进账户逻辑。
- Bundler：打包，负责收集 UserOperation，模拟验证（如判断操作是否有效）并提交到 EntryPoint。
- Paymaster：允许第三方为用户操作支付 gas，或者让用户用非原生资产承担费用。Paymaster 常用于 onboarding（新用户完成第一次操作）。
- Session Key：给应用或 Agent 的临时权限，

**Account Abstraction 与 AI x Web3**

Account Abstraction 是 AI Agent 上链执行的重要底座。没有账户抽象，Agent 往往只能停留在“给建议”或“让用户每一步都签名”。有了智能账户、Paymaster 和 Session Key，Agent 才可能在受限范围内自动执行。

### DeFi去中心化金融

> DeFi 协议管理的是资产状态，不只是应用界面。

DeFi 是建立在智能合约上的开放金融系统。它让资产、交易、借贷、稳定币和流动性可以被协议化组合，也让风险沿着协议依赖链传播。

- 资产是协议输入：token 标准、decimals、授权和余额都会影响协议行为。
- 流动性决定可执行性：有价格不等于能成交，能成交不等于低成本成交。
- 可组合性放大风险：一个协议出问题，可能影响依赖它的多个协议。

**知识点**：

- Token 是 DeFi 的资产单位，但不同 token 的标准、精度、权限和风险差异很大。ERC-20 是最常见的同质化 token 标准。用户余额、转账、授权、交易和借贷通常都围绕 token 展开。错误的 token 地址和 decimals（代币的小数位数）处理，是 DeFi 前端和 Agent 工具里非常常见的风险源。
- AMM：Automated Market Maker，自动做市商。用流动性池和定价公式替代传统订单簿，让用户可以直接和合约交易。用户把两种资产存入池子，交易者用一种资产换另一种资产，价格由池子状态和公式决定。流动性越深，同样规模交易的价格冲击通常越小。AMM 里最重要的几个概念：
  - 滑点：预期价格和实际成交价格的差距。
  - 流动性提供者：向池子提供资产，赚取手续费，也承担无常损失。
  - 价格影响：大额交易会改变池子比例，导致成交价格变差。
  - MEV 风险：交易排序可能被利用，例如 sandwich attack。
- Lending：借贷协议把存款、借款、抵押率、利率和清算规则写进合约。借款能否维持，取决于抵押品价值、债务价值、清算阈值和预言机价格。
- Stablecoin：稳定币，DeFi 里的计价和结算基础。常见稳定币可能由法币储备、加密资产超额抵押、算法机制或混合结构支撑。
- Liquidity：流动性，决定资产能不能以合理价格买入、卖出、借出、清算或退出。

**DeFi 与 AI x Web3**

AI 进入 DeFi，最适合先做信息整理和风险辅助：解释交易、总结仓位、监控清算、识别异常价格、生成策略草案。真正高风险的是自动执行：swap、借贷、加杠杆、授权、跨链和清算。

如果 Agent 要执行 DeFi 操作，至少要有：额度限制、协议白名单、滑点上限、模拟结果、预言机检查、人工确认和交易后审计记录。
