# 260530 学习笔记

## AI × Web3 Bridge

### 智能体钱包（Agent Wallet）

Agent Wallet 的核心不是让 AI 完全接管私钥，而是让用户能把**可撤销、可审计的有限权限**交给 Agent，让其在用户预设的边界内安全执行链上操作。

#### 核心知识点

* Account Abstraction (AA 钱包)：让账户本身具备规则表达能力，而非仅由私钥控制。可将签名、权限、支付 Gas、恢复、多签、额度、session key 等逻辑内置到账户规则中，为 Agent 提供可控的执行边界。
* Smart Account：带规则的链上账户，定义谁可发起操作、哪些操作需人工确认、可调用合约、单日消费上限、权限有效期、异常暂停等规则，承载 Agent 自动化执行的边界。
* Safe：多签智能账户基础设施，天然适合团队 / DAO 场景，Agent 可生成交易草稿或执行低风险模块，关键转账 / 合约升级仍需多签成员确认，避免单 Agent 控制全部资产。
* Session Key：给 Agent 分配的临时、有限权限密钥，需限制有效期、可调用合约 / 方法、单笔 / 总额度、可操作资产类型，支持用户随时撤销，平衡自动化与安全性。
* Policy：Agent 执行链上动作时必须遵守的可检查规则，如单日消费上限、仅可调用白名单合约、滑点限制、高风险操作必须人工确认，明确界定 Agent 执行的边界。
* Guard：交易前的确定性主拦截层，对 Agent 生成的交易做最终规则校验，检查目标地址白名单、调用方法、金额超限、价格滑点、交易模拟结果等，拦截不符合策略的操作。
* Simulation：在签名和广播前预演交易结果，提前验证资产变化、Gas 成本、合约执行结果，将 calldata 翻译成用户可理解的语言，降低错误执行风险。
* Revocation：关注权限的收回机制，用户可随时撤销 Agent 权限，系统也可在 session key 到期、额度用尽、多次交易失败、用户长时间未确认时自动收回权限，所有自动化权限需提供可见的关闭入口。
* Human Check：按风险分层的人工确认机制，低风险操作自动执行，中风险操作需用户确认，高风险 / 超出策略的操作直接拒绝，用户需能清晰看到交易的资产变化、风险点再决定是否确认。

### 在 AI × Web3 中的位置

Agent Wallet 是 AI × Web3 的关键连接层：AI 负责理解目标、生成计划；
Agent Wallet 负责执行链上动作，但不直接触碰用户资产控制权。
