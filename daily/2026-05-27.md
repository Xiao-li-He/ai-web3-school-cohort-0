## Notes

- `decimals` 是 token 链上整数金额和人类可读金额之间的小数位换算规则。
- ERC-20 token 的余额通常以整数存储，前端和工具需要用 `decimals` 做格式化和解析。
- 不同 token 的 decimals 可能不同：USDC / USDT 常见为 6，ETH / WETH / DAI 常见为 18，WBTC 常见为 8。
- AMM 是 Automated Market Maker，用流动性池和定价公式替代传统订单簿。
- 用户在 AMM 里不是和某个具体对手方交易，而是和智能合约里的流动性池交易。
- AMM 交易需要关注滑点、手续费、池子深度、交易路径、token 地址和 decimals。

## Agent DeFi Preflight Checklist

Before an Agent prepares or explains an AMM swap, check:

- Source chain and target chain.
- Token contract addresses, not only token symbols.
- Token decimals for input and output assets.
- User-facing amount after formatting.
- Raw integer amount used in transaction calldata.
- Pool liquidity and expected output.
- Slippage tolerance.
- Fee tier or protocol fee.
- Swap route and intermediate tokens.
- Existing allowance and approval amount.
- Whether approval is exact amount or unlimited.
- Recipient address.
- Deadline or expiration.
- Estimated gas and native token balance.
- Transaction simulation result, if available.
- Explorer links for token, pool, and transaction after submission.

Actions that require explicit user confirmation:

- Token approval.
- Unlimited allowance.
- Swap execution.
- Interacting with unknown token contracts.
- Using a route with low liquidity or high slippage.
- Any transaction where displayed amount and raw amount do not match expected decimals.

今天继续学习 DeFi 基础，重点理解了 token decimals 和 AMM。

我的理解是：链上的 token 金额通常用整数存储，`decimals` 决定这个整数如何转换成人类可读金额。不同 token 的 decimals 不一定相同，例如 USDC 常见是 6，ETH/WETH 常见是 18。如果前端或 Agent 工具把 decimals 处理错，可能会导致金额被放大或缩小很多倍，这是 DeFi 工具里非常典型的风险源。

AMM 是自动做市商，用流动性池和公式进行定价，不依赖传统订单簿。用户交易时是和合约里的池子交易，因此需要检查流动性、滑点、手续费、交易路径、token 地址和授权额度。

结合 AI x Web3，我整理了一个 Agent DeFi 预检查清单。后续如果 Agent 要帮助用户准备 swap，至少要先确认 token 地址、decimals、raw amount、显示金额、预期输出、滑点、allowance、recipient 和交易模拟结果。涉及 approval、无限授权、swap 执行、未知 token 或高滑点时，必须要求用户明确确认。
