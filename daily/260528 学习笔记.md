# 260528 学习笔记

## AI × Web3 Bridge

---

### 链感知上下文（Chain aware Context）

合约和链上数据会持续变化，AI Agent 必须基于**实时、可验证的链上事实**做出决策，而不是依赖模型记忆或过时信息。链感知上下文就是为 AI 提供可靠链上事实的基础层。

#### 核心知识点

- **On chain Data**：余额、交易日志、合约状态、区块信息等可直接验证的链上数据，读取时必须携带 `chain id`、区块号、合约地址等溯源字段。

- **Contract Docs; ABI/Event**：ABI 仅提供函数签名，需结合文档理解业务逻辑；Event 是合约对外的业务日志（如 Transfer），是 AI 理解链上行为的关键入口。

- **Transaction; Explorer Context**：交易历史需保留哈希、区块号等信息，区块浏览器链接提供可视化证据，让用户可核验交易状态与合约验证情况。

- **Indexing; Citation**：索引层将链上事件整理为可查询数据（如用户历史 DeFi 操作）；Citation 为模型结论标注链上来源，确保解释可验证、可追溯。

---

### Web3 工具调用（Web3 Tool Use）

AI Agent 进入 Web3 不能只靠文本生成，必须通过工具读写链上数据、发起交易。Web3 工具的核心是让每一次链上动作都有**结构化输入、明确权限和可审计结果**。

#### 核心知识点

- **RPC Tool**：用于读取链状态、查询区块、估算 Gas，返回结果需包含链 ID、Provider、区块号，确保数据来源可追溯。

- **Contract Read/Write**：`Read` 调用合约 view/pure 函数，不改变链状态；`Write` 会修改链状态，必须经过参数校验、模拟执行、用户确认，且仅允许白名单合约 / 方法。

- **Wallet; DeFi Tool**：钱包工具需拆分连接、签名、发送交易等动作，禁止 AI 直接绕过钱包确认；DeFi 工具需设置协议白名单、最大交易额 / 滑点、人工确认策略，避免资产风险。

- **Tool Permission \&amp; Log**：按工具、合约、金额、用户确认级别分层权限；每次调用记录输入、输出、错误、时间等日志，出错时可完整还原执行链路。

---

### 智能体工作流（Agent Workflow）

链上操作有资产、权限和不可逆结果，AI Agent 不能自由发挥，必须通过工作流将模型推理约束在**可控、可验证的流程**中，核心是把概率模型放进确定性流程。

#### 核心知识点

- **Task Graph; State Machine**：将目标拆分为节点和依赖（如 swap 拆分为读取目标、获取余额、模拟交易、用户确认等步骤）；用状态机（draft、submitted、confirmed 等）记录执行进度，避免重复执行危险动作。

- **Human\-in\-the\-loop; Retry/Fallback**：关键风险点引入人工确认（如高风险交易），低风险步骤可自动执行；工具失败时合理重试（如余额读取失败可重试），避免盲目重发交易。

- **Trace; Regression Set**：Trace 记录每一步输入、判断、工具调用和结果，便于复盘；Regression Set 是固定测试用例（如正常 swap、错误链请求、用户拒绝签名），防止模型更新后安全性退化。

- **Evaluation Harness**：系统测试 Agent 在不同任务、风险场景下的表现，验证其是否能拒绝越权请求、识别错误合约、规避危险 calldata。
