## Notes

- Wallet 不是资产本身，而是管理账户控制权、发起签名和确认交易的入口。
- 连接钱包通常只代表应用可以读取地址、网络等公开上下文，不代表应用可以移动资产。
- 签名消息用于证明地址控制权或表达授权意图；发送交易会改变链上状态，风险更高。
- Transaction 可能是普通转账，也可能是合约调用。用户需要理解目标链、合约地址、参数、gas 和潜在资产变化。
- Explorer 是验证链上结果的重要工具，适合用于检查交易状态、合约、事件和 token 转移。

## AI x Web3 Permission Boundary Draft

An AI Agent can:

- Read public chain data.
- Explain wallet prompts and transaction intent.
- Prepare transaction parameters for user review.
- Estimate risk and summarize possible asset changes.
- Record trace logs for later review.

An AI Agent must ask for human confirmation before:

- Requesting a wallet signature.
- Sending or broadcasting a transaction.
- Approving token spending.
- Switching networks for a user action.
- Interacting with an unknown or high-risk contract.

An AI Agent must never:

- Ask for seed phrases, private keys, or raw wallet secrets.
- Hide transaction details from the user.
- Treat wallet connection as permission to spend assets.
- Auto-confirm irreversible or value-moving actions.
