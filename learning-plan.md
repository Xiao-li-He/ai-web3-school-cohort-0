# Learning Plan

Updated: 2026-05-18

## Goal

Prepare for AI x Web3 development and Hackathon work by building enough Web3
foundation to design, prototype, and explain a small agent-related product.

## Daily Rhythm: 60 Minutes

1. 10 minutes: review today's WCB Learning page and pick the smallest task.
2. 25 minutes: read one Handbook section or course material.
3. 15 minutes: make one concrete artifact, such as notes, a script, a diagram,
   a task write-up, or a tiny prototype.
4. 10 minutes: write the daily note, check-in draft, and any Handbook feedback.

## Week 1: Web3 Foundation for AI Builders

- Read: Network, Cryptography, Wallet, Smart Contract.
- Practice: write short explanations for accounts, signatures, transactions,
  RPC, gas, and contract calls.
- Artifact: one beginner-friendly glossary in `tasks/`.

## Week 2: Agent + Web3 Tool Use

- Read: Agent, MCP, Web3 Tool Use, Agent Workflow.
- Practice: map an agent workflow with human confirmation before risky actions.
- Artifact: a toy tool-calling workflow design in `experiments/`.

## Week 3: Wallet, Permissions, and Safety

- Read: Account Abstraction, Agent Wallet, AI Security, AI Privacy.
- Practice: describe what an agent is allowed to do, what it must ask for, and
  what must be logged.
- Artifact: a permission policy draft for a Hackathon idea.

## Week 4: Hackathon Prototype Direction

- Read: Chain-aware Context, Agentic Commerce, Dev Tooling, Wallet / Permission.
- Practice: choose one narrow user problem and scope a demo.
- Artifact: `hackathon/project-brief.md` with problem, users, flow, risks, and demo plan.

## Default Today

- Minimum path: read Handbook homepage and Web3 foundation outline.
- Recommended path: read Wallet and Smart Contract pages if time allows.
- Challenge path: write a one-page idea for an agent that reads chain state but
  cannot sign transactions without human confirmation.
